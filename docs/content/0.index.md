## Welcome to the WiCAN docs.

Use the menu to browse the documentation. Also check out the [contributing](/contributing) page if you want to help add to or update these docs.

These docs assume you are running the latest version of the WiCAN firmware, so please keep your firmware up to date.

These docs are still new and very much a work in progress. If you find useful content in the main README, GitHub issues, or discussions, feel free to help move it here so it is easier to find in one central place.

## Frequently Asked Questions

**Q: Can I buy locally in Australia?**  
A: We are an Australian small business. At the moment, WiCAN and WiCAN PRO are sold exclusively through [Mouser](https://www.mouser.com/c/?m=MeatPi) and [Crowd Supply](https://www.crowdsupply.com/meatpi-electronics). We do not have the logistics in place to ship individual orders directly. [Mouser](https://www.mouser.com/c/?m=MeatPi) also offers free shipping to Australia and some other countries.

**Q: Should I buy [WiCAN PRO](https://www.meatpi.com/products/wican-pro) or [WiCAN](https://www.meatpi.com/products/wican)?**  
A: The original WiCAN started as a basic adapter. WiCAN PRO brings together the lessons learned from that first version, plus community feedback and the features users asked for most. If you want the best overall experience and value, WiCAN PRO is the recommended option.

**Q: Is my Car model supported?**  
A: WiCAN PRO supports most modern vehicles that use OBD2. 

- If your car is **NOT** electric, you can usually scan for available PIDs in WiCAN PRO auto mode and add the parameters that are found to Home Assistant.

- If your car is electric, check the [supported vehicles list](/config/automate/supported_vehicles) first. If your model is not listed, open an issue on GitHub and support can be added. The [supported vehicles list](/config/automate/supported_vehicles) mainly covers cars, especially EVs, that have extra vehicle-specific parameters.
